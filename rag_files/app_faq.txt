# FAQ - LLM Chat Application

## Что такое LLM Chat?
LLM Chat - это консольное приложение на Kotlin для общения с Claude AI от Anthropic.
Приложение поддерживает стриминг ответов, RAG (поиск по документам), MCP инструменты
и множество других функций для продуктивной работы с AI.

## Есть ли в приложении авторизация?
Нет, в приложении нет системы авторизации. Это локальное консольное приложение,
которое работает на вашем компьютере. Никакой регистрации или входа не требуется.
Все данные (история чатов, настройки) хранятся локально в SQLite базе данных
на вашем компьютере в папке ~/.llm-chat/.

## Нужна ли регистрация для использования?
Нет, регистрация не нужна. Единственное что требуется - это API ключ от Anthropic,
который вы получаете на сайте console.anthropic.com. Этот ключ нужен для
взаимодействия с AI моделями Claude.

## Как запустить приложение?
1. Клонируйте репозиторий
2. Установите переменную окружения ANTHROPIC_API_KEY:
   export ANTHROPIC_API_KEY=your_key_here
3. Выполните команду: ./gradlew run
4. Введите сообщение в консоли и нажмите Enter

## Какие модели поддерживаются?
Приложение использует модели Claude от Anthropic:
- Claude Sonnet 4 (claude-sonnet-4-20250514) - основная модель для диалогов
- Claude Haiku 4.5 (claude-haiku-4-5-20251001) - быстрая модель для инструментов и сжатия
- Claude Opus 4.1 - поддерживается для сложных задач

## Где получить API ключ?
API ключ можно получить на сайте console.anthropic.com после регистрации.
После получения ключа установите его в переменную окружения ANTHROPIC_API_KEY.

## Консольные команды приложения

### Основные команды:
- exit - выход из приложения
- /new или /clear - начать новый диалог (очистить историю)
- /stats - показать статистику текущей сессии
- /help - помощь по кодбазе проекта

### Настройки модели:
- /changePrompt - сменить системный промпт (персону AI)
- /temperature <0.0-1.0> - изменить температуру генерации
- /maxTokens <число> - изменить максимальное количество токенов в ответе

### RAG команды:
- /rag status - показать статус индекса RAG
- /rag index - проиндексировать файлы из rag_files/
- /rag reindex - переиндексировать все файлы заново
- /rag search <запрос> - поиск по базе знаний
- /rag on - включить RAG контекст в запросах
- /rag off - выключить RAG контекст
- /rag debug - показать полный запрос с RAG контекстом
- /rag compare <запрос> - сравнить результаты с реранкингом и без
- /rag threshold <0-1> - настроить порог релевантности
- /rag reranker on/off - управление реранкингом
- /rag method <метод> - выбрать метод реранкинга

### MCP команды:
- /mcp - показать подключенные MCP серверы и доступные инструменты
- /mcp connect <server> - подключить MCP сервер
- /mcp disconnect <server> - отключить MCP сервер

### Управление памятью:
- /memory - показать сохраненные сообщения
- /memory clear - очистить память

## Что такое RAG?
RAG (Retrieval-Augmented Generation) - это технология, которая позволяет
AI находить и использовать релевантную информацию из ваших документов
при генерации ответа. Это делает ответы более точными и контекстно-зависимыми.

## Что такое MCP?
MCP (Model Context Protocol) - это протокол для расширения возможностей AI
с помощью внешних инструментов. Например, AI может искать в Wikipedia,
сохранять файлы, работать с Git и GitHub, управлять Android эмулятором.

## Как работает стриминг?
Приложение использует Server-Sent Events (SSE) для получения ответов от API.
Это позволяет видеть ответ по мере его генерации, а не ждать полного ответа.

## Какие системные промпты доступны?
Приложение предлагает несколько предустановленных персон:
- Технический аналитик для создания ТЗ
- Логический решатель задач
- Технический писатель
- И другие (используйте /changePrompt для просмотра списка)

## Как добавить свой системный промпт?
Отредактируйте файл src/main/kotlin/utils/SystemPrompts.kt
и добавьте свой промпт в список AVAILABLE_PROMPTS.

## Поддерживается ли работа с несколькими LLM провайдерами?
Да, архитектура приложения поддерживает множественные LLM провайдеры
через единый интерфейс LlmClient. Помимо Anthropic поддерживается OpenRouter.

## Как сохраняется история диалога?
История сохраняется в SQLite базе данных. При достижении лимита сообщений
происходит автоматическое сжатие истории с помощью Claude Haiku для экономии токенов.
